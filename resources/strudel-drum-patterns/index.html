2 Simple sequences

You can play a longer sequence as shown below. By default the pattern will be scaled to play over two seconds, so the more beats you add in, the fast things move.

Try changing the sequence. There are a variety of default drum sounds that you can use for this:

  • bd = bass drum
  • sd = snare drum
  • rim = rim shot
  • cp = clap
  • hh = closed hi-hat
  • oh = open hi-hat
  • cr = crash
  • rd = ride
  • ht = high tom
  • mt = mid tom
  • lt = low tom
  • - = [rest]
You can read more about these drum sounds in the Strudel documentation

3 Expanding sequences

We can be a little more advernturous with our patterns. Below are a range of techniques for that.

3.1 Repeats with *X

You can repeat a particular sound or event by "multiplying it".

3.2 Expanding particular beats with [ ... ]

Placing events inside the square brackets groups them together as a single beat. This can mean that the beats play faster, e.g. squeezing 2, 3 or more beats into the time of a single beat.

3.3 Alternating particular beats with < ... >

If you use the angular brackets < ... > instead, rather than changing the timing, the pattern will alternate between the different elements each time that the sequence repeats

3.4 Changing the rate of things by wrapping everything in <> brackets

Currently everything is squeezed into a 2 second loop. We can have a little more control over this by wrapping everything in the <> brackets, and then using the *X as shown above to increase or decrease the speed. Try changing the *8 in the example below.

3.5 Reusing the same techniques for the speed

One of the lovely things about strudel is that any of the pattern making processes you learn in one place can usually be applied in other places. For instance, we can make the speed change alternate using the <> the same way we did above for individual beats.
